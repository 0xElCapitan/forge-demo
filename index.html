    How it works

    Ingest, characterize, serve.

    The pipeline is deliberately narrow. FORGE owns the feed-to-envelope path. Echelon owns admission, instantiation, market operation, and settlement.

    01 / Ingest

    Normalize the feed into deterministic events.

    Fixtures and live adapters land in the same shape first. That keeps downstream classification auditable and keeps the boundary between adapter logic and oracle logic clean.

    JSON fixture or live adapter input Normalized event array Source metadata attached once
    02 / Characterize

    Pythia classifies the signal across five dimensions.

    Cadence, distribution, noise, density, and thresholds produce the feed profile. That profile is the machine readable truth of the feed, not a guess at its story.

    FeedProfile generated in under 50ms Deterministic selector input Useful even when natural language is inadequate
    03 / Serve

    Themis emits the right theatre template to the seam.

    The output is a ProposalEnvelope, not a running market. That keeps FORGE data-pure while giving Echelon a stable, typed contract to admit, reject, or benchmark.

    threshold_gate through persistence Deterministic proposal_id scope Echelon admission gate consumes the envelope
